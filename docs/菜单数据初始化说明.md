# 菜单数据初始化说明

## 问题说明

后端服务重启后，新添加的菜单（法律法规、规范用语、监管事项）没有显示在界面上。

**原因**：菜单数据 SQL 文件已创建，但尚未执行到数据库中。

## 解决方法

### 1. 启动 MySQL 数据库

```bash
# 如果使用系统 MySQL
brew services start mysql  # macOS
# 或
sudo systemctl start mysql  # Linux

# 如果使用 Docker
docker-compose up -d mysql
```

### 2. 执行菜单 SQL 文件

```bash
# 方式一：使用完整菜单 SQL（推荐）
mysql -h localhost -u root -p haocean_health < config/sql/law_menu_full.sql

# 方式二：只执行扩展菜单（如果已有基础菜单）
mysql -h localhost -u root -p haocean_health < config/sql/law_menu_extend.sql
```

### 3. 重启后端服务（可选）

```bash
# 后端会自动从数据库加载最新菜单数据
go run main.go
```

### 4. 刷新浏览器

访问 http://localhost:8080/admin，刷新页面后应该能看到新增的菜单：

```
卫生执法
├── 行业分类
├── 监管单位
├── 执法人员
├── 设备管理
├── 激活码
├── 文书模板
├── 执法记录
├── 数据同步
├── 法律法规（新增）
├── 规范用语（新增）
└── 监管事项（新增）
```

## 常见问题

### 问题 1：执行 SQL 报错 "Duplicate entry"

**解决**：SQL 文件中已包含 `ON DUPLICATE KEY UPDATE`，如果仍报错，可以先删除旧数据：

```sql
-- 删除旧菜单（谨慎操作！）
DELETE FROM sys_menu WHERE menu_id >= 200 AND menu_id <= 211;
-- 重新执行 SQL
source config/sql/law_menu_full.sql;
```

### 问题 2：菜单显示但点击无反应

**解决**：
1. 检查前端路由配置是否正确
2. 清除浏览器缓存
3. 检查浏览器控制台是否有错误信息

### 问题 3：菜单不显示

**解决**：
1. 确认 SQL 已执行成功
2. 检查数据库中是否有菜单数据：`SELECT * FROM sys_menu WHERE parent_id = 200;`
3. 刷新浏览器或重新登录
4. 检查用户角色是否有菜单权限
