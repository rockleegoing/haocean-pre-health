# 移动卫生执法系统 - 开发流程规范

> 开发任何功能前必须严格按此流程执行

---

## 标准开发流程（5 步）

### 步骤 1：阅读 PRD.md 产品需求
**目的**: 理解功能的业务目标、用户需求、数据流向

**操作**:
1. 打开 `docs/PRD.md`
2. 搜索对应功能章节（如"快速拍照"、"法律法规"）
3. 记录关键信息：
   - 功能描述
   - 用户操作流程
   - 数据结构要求
   - 接口定义（如有）
   - 离线/同步要求

**输出**: 功能需求要点（用于后续设计参考）

---

### 步骤 2：阅读原型图 + 原型图分析.md
**目的**: 明确页面 UI 布局、交互细节、字段定义

**操作**:
1. 打开 `docs/原型图/` 对应图片
2. 阅读 `docs/原型图分析.md` 对应章节
3. 记录关键信息：
   - 页面标题、导航栏样式
   - 所有输入/输出字段
   - 按钮操作及跳转逻辑
   - 弹窗/选择器等交互组件
   - 数据展示格式

**输出**: 页面结构设计（组件列表、字段映射）

---

### 步骤 3：查阅相关技术文档
**目的**: 找到可复用的代码、组件、API

**操作**:
1. 阅读 `docs/MOBILE.md` - 移动端开发规范
2. 查阅 uni-app 官方文档：
   - 组件：https://uniapp.dcloud.net.cn/component/
   - API: https://uniapp.dcloud.net.cn/api/
3. 查阅 uView UI 文档：
   - https://www.uviewui.com/components/
4. 查找现有代码中可复用的模块：
   - 通知公告模块（列表/详情/编辑）
   - 用户管理模块（CRUD 模式）

**输出**: 技术方案设计（复用组件 + 新增组件）

---

### 步骤 4：选择适合的开发工具
**目的**: 使用正确的插件/Skill 提高开发效率

**可用工具**:

| 工具 | 用途 | 选择条件 |
|------|------|---------|
| `/astock-fullstack` | A 股研报生成器专属开发 | 涉及后端 API 开发 |
| `Agent (Explore)` | 代码库探索 | 查找相似功能、可复用代码 |
| `Agent (Plan)` | 实现方案设计 | 复杂功能需要设计方案 |
| `TaskCreate` | 任务跟踪 | 多步骤功能开发 |

**选择原则**:
- 优先使用现有代码复用（Agent Explore 查找）
- 复杂功能先设计方案（Agent Plan）
- 多步骤任务用 TaskCreate 跟踪

---

### 步骤 5：正式开发执行
**目的**: 按设计实现功能

**开发顺序**:
1. **后端优先**（如需要）:
   - Model 层（数据模型）
   - Controller 层（API 接口）
   - Router 层（路由配置）
   - Service 层（业务逻辑）

2. **移动端开发**:
   - 创建页面文件（.vue）
   - 编写模板（template）
   - 实现逻辑（script）
   - 添加样式（style）
   - 集成 API 调用

3. **联调测试**:
   - 启动后端：`go run main.go`
   - 启动前端：`npm run dev`
   - 浏览器调试 H5

---

## 开发检查清单

开发每个功能前，请确认已完成以下步骤：

```
□ 已阅读 PRD.md 对应章节
□ 已查看原型图（所有相关图片）
□ 已阅读原型图分析.md 对应章节
□ 已查阅 uni-app/uView 文档
□ 已查找现有代码可复用部分
□ 已确定技术方案
□ 已创建 Task（如需要多步骤）
```

## 示例：开发"快速拍照"功能

### 步骤 1: 阅读 PRD
```
PRD.md → 5.2.1 便捷执法功能 → 1. 快速拍照
需求要点:
- 调用相机拍照
- 支持连续拍照
- 照片自动关联单位和行业
- 保存到执法记录
```

### 步骤 2: 阅读原型图
```
原型图/便捷执法_快速拍照.png
原型图分析.md → 3.2 快速拍照
页面要素:
- 入口：首页"快速拍照"按钮
- 交互：调用相机 → 拍照 → 保存
- 字段：单位、行业、检查日期、执法人员
```

### 步骤 3: 查阅文档
```
uni-app API → uni.chooseImage()
uView UI → u-upload 组件
现有代码 → 无直接复用，参考 notice/edit.vue 表单结构
```

### 步骤 4: 选择工具
```
使用 TaskCreate 创建任务（多步骤）
使用 Agent Explore 查找类似图片上传功能
```

### 步骤 5: 开发执行
```
1. 创建页面 pages/enforcement/photo.vue
2. 实现拍照功能 uni.chooseImage()
3. 保存逻辑（本地 SQLite + 文件存储）
4. 后端接口 POST /api/evidence/photo
```
